# Roadmap Produit — Module FLOW

> **FLOW n'est pas un projet ponctuel.**
>
> Il s'agit d'un module fondateur de MOMENTUM qui évoluera progressivement au fil des versions.
> Cette feuille de route décrit la vision d'évolution du module sans imposer un calendrier figé.

---

# Vision

Le module FLOW a pour ambition de devenir le moteur de compréhension des expériences vécues.

Aujourd'hui, il représente les activités.

Demain, il aidera l'utilisateur à comprendre :

- ce qui le stimule ;
- ce qui le fait progresser ;
- ce qui le ressource ;
- dans quelles conditions il vit ses meilleurs moments.

Chaque évolution devra conserver un principe fondamental :

> **Le ressenti de l'utilisateur reste la référence.**

Les analyses automatiques viennent enrichir l'expérience, jamais la remplacer.

---

# Version 1.0

## Objectif

Créer les fondations du module FLOW.

## Fonctionnalités

- Graphique Défi × Maîtrise.
- Trois questions après l'activité :
  - Effort physique
  - Défi
  - Maîtrise
- Représentation des activités sous forme de points.
- Zones du Flow.
- Résumé interactif d'une activité.
- Explication du concept de Flow.
- Exploitation des données FIT comme contexte.
- Architecture technique évolutive.

---

# Version 1.5

## Objectif

Améliorer la qualité des analyses sans modifier l'expérience utilisateur.

## Évolutions

- Exploitation plus poussée des données FIT.
- Analyse de la charge aiguë (ATL).
- Analyse de la charge chronique (CTL).
- Analyse du TSB.
- Analyse de la dérive cardiaque.
- Analyse des zones cardiaques.
- Analyse des conditions environnementales.

Ces informations enrichissent le contexte de chaque activité.

Elles ne modifient toujours pas automatiquement la position des points.

---

# Version 2.0

## Objectif

Faire apparaître l'intelligence de MOMENTUM.

## Évolutions

MOMENTUM commence à reconnaître des schémas.

Exemples :

- activités les plus engageantes ;
- sports générant le plus souvent du Flow ;
- périodes favorables ;
- évolution de la maîtrise.

L'utilisateur commence à recevoir des observations.

Exemple :

> Les sorties trail entre 2h et 3h génèrent fréquemment des expériences proches du FLOW.

Aucune recommandation automatique n'est encore proposée.

---

# Version 3.0

## Objectif

Créer la FLOW MAP.

Le graphique évolue vers une lecture beaucoup plus riche.

Exemples d'analyses :

- répartition géographique ;
- saisons ;
- météo ;
- partenaires d'activité ;
- disciplines ;
- durée ;
- intensité.

L'utilisateur commence à visualiser où et dans quelles conditions il vit ses meilleures expériences.

---

# Version 4.0

## Objectif

Faire de FLOW un compagnon de décision.

À partir de l'historique personnel, MOMENTUM pourra proposer :

- des idées d'activités ;
- des périodes favorables ;
- un niveau de défi adapté ;
- un équilibre entre récupération, progression et découverte.

Ces suggestions restent des recommandations.

L'utilisateur garde toujours le dernier mot.

---

# Version 5.0

## Objectif

Comprendre le parcours de vie.

FLOW ne se limite plus au sport.

Il peut progressivement intégrer :

- bien-être ;
- aventures ;
- découvertes ;
- moments partagés ;
- projets personnels.

Le module devient un journal visuel des expériences marquantes.

---

# Principes d'évolution

Toutes les futures évolutions devront respecter les principes suivants.

## Le ressenti est prioritaire

Les données objectives enrichissent.

Le ressenti décide.

---

## Les données restent intactes

Les algorithmes évoluent.

Les données utilisateur ne sont jamais modifiées.

---

## Transparence

L'utilisateur doit toujours savoir :

- ce qu'il a déclaré ;
- ce que MOMENTUM calcule.

---

## Pas de notation

FLOW ne note jamais l'utilisateur.

FLOW raconte son parcours.

---

## Pas de zone idéale permanente

Le FLOW n'est pas un objectif quotidien.

Explorer, apprendre, récupérer, se dépasser ou simplement vivre une période de relaxation font tous partie d'un parcours équilibré.

---

# Vision à long terme

À terme, le module FLOW devra devenir la signature de MOMENTUM.

Les autres plateformes racontent :

> **Ce que tu as fait.**

FLOW racontera :

> **Comment tu l'as vécu.**

C'est cette différence qui doit guider chacune des évolutions futures.

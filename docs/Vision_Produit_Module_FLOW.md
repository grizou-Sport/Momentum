# Vision Produit — Module FLOW

> *Les données racontent ce que nous avons fait. Le FLOW raconte ce que nous avons vécu.*

---

# Pourquoi FLOW existe

La plupart des plateformes sportives savent mesurer la performance :

- kilomètres ;
- dénivelé ;
- fréquence cardiaque ;
- puissance ;
- charge ;
- récupération.

Ces informations sont essentielles.

Mais elles ne répondent pas à la question la plus importante :

> **Comment cette expérience a-t-elle été vécue ?**

Deux activités identiques peuvent produire deux expériences totalement différentes.

Une sortie parfaite peut être oubliée.

Une sortie imparfaite peut devenir un souvenir inoubliable.

FLOW est né pour raconter cette différence.

---

# Notre conviction

Nous croyons que les expériences qui nous construisent ne sont pas uniquement les plus difficiles.

Ce sont celles où le défi rencontré rejoint notre sentiment de maîtrise.

C'est dans cet équilibre que naissent :

- l'engagement ;
- la concentration ;
- l'apprentissage ;
- le plaisir ;
- parfois même la sensation de perdre la notion du temps.

Cet état porte un nom.

# FLOW

---

# Ce que FLOW n'est pas

FLOW n'est pas un score.

FLOW n'est pas une note.

FLOW n'est pas un classement.

FLOW n'est pas un objectif quotidien.

Le but n'est pas de placer tous les points dans la zone FLOW.

Chaque zone représente un moment différent du parcours.

- Explorer
- Apprendre
- Récupérer
- Routine
- Relaxation
- Maîtriser
- Se dépasser
- FLOW

Aucune zone n'est mauvaise.

Toutes racontent une expérience différente.

---

# Ce que FLOW apporte

FLOW transforme une succession d'activités en une histoire.

Chaque point devient le souvenir d'un moment.

Progressivement, l'utilisateur découvre :

- quelles activités le stimulent ;
- lesquelles le mettent en difficulté ;
- lesquelles deviennent routinières ;
- lesquelles favorisent son engagement.

Le graphique ne cherche pas à dire :

> **Tu progresses.**

Il cherche à montrer :

> **Voilà comment tu vis ton chemin.**

---

# Les données au service de l'humain

MOMENTUM exploite de nombreuses données objectives :

- fréquence cardiaque ;
- charge d'entraînement ;
- puissance ;
- durée ;
- dénivelé ;
- météo ;
- fatigue ;
- données FIT.

Ces données servent à comprendre le contexte.

Elles ne remplacent jamais le ressenti.

Le ressenti de l'utilisateur reste la référence.

L'algorithme accompagne.

Il ne décide pas.

---

# Une représentation vivante

Chaque activité devient un point.

Chaque point raconte une expérience.

En cliquant sur un point, l'utilisateur retrouve :

- son activité ;
- son contexte ;
- son ressenti ;
- ses notes ;
- ses souvenirs.

Le graphique devient progressivement une carte personnelle des expériences vécues.

---

# Notre ambition

Aujourd'hui, FLOW est un graphique.

Demain, il deviendra une carte.

Puis un compagnon capable d'aider chacun à mieux comprendre les conditions dans lesquelles il se sent engagé, vivant et aligné avec lui-même.

Cette évolution se fera progressivement.

Sans jamais perdre un principe fondamental :

> **Le rôle de MOMENTUM n'est pas de dire à l'utilisateur ce qu'il devrait ressentir. Il est de l'aider à comprendre ce qu'il a réellement vécu.**

---

# La promesse du module FLOW

Le module FLOW ne mesure pas uniquement une activité.

Il révèle la relation entre l'être humain et l'expérience qu'il traverse.

Parce qu'au bout du compte, ce ne sont pas les kilomètres, les watts ou les chronos que nous retenons.

Ce sont les moments où nous nous sommes sentis pleinement présents.

Et c'est précisément ce que FLOW cherche à préserver.

---

# Notre signature

> **Les performances s'oublient. Les expériences nous transforment. FLOW existe pour comprendre celles qui comptent vraiment.**

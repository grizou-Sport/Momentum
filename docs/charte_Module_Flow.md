# Charte Scientifique — Module FLOW

> *Le rôle de FLOW n'est pas de juger les expériences.
> Il est de les comprendre.*

---

# Préambule

Le module FLOW est inspiré des travaux du psychologue Mihály Csíkszentmihályi sur l'état de Flow.

MOMENTUM ne cherche pas à reproduire un outil académique.

Nous souhaitons adapter ce modèle afin qu'il accompagne durablement les utilisateurs dans leur pratique sportive, leur bien-être et leurs aventures.

Cette adaptation devra toujours respecter les principes décrits dans ce document.

---

# 1. Le ressenti est la référence

Le ressenti de l'utilisateur constitue la donnée la plus importante du module FLOW.

Aucune donnée automatique ne doit remplacer ce ressenti.

Les données physiologiques servent à comprendre le contexte.

Elles ne doivent jamais corriger ce que l'utilisateur déclare avoir vécu.

---

# 2. Les données enrichissent, elles ne décident pas

Les données issues :

- du fichier FIT ;
- de la fréquence cardiaque ;
- de la puissance ;
- de la charge ;
- du sommeil ;
- de la récupération ;
- de la météo ;
- ou de tout autre capteur,

sont utilisées pour enrichir l'analyse.

Elles ne définissent jamais seules l'expérience vécue.

---

# 3. Aucune zone n'est meilleure qu'une autre

Le FLOW n'est pas une récompense.

Les autres zones ne sont pas des échecs.

Chaque zone raconte une étape différente du parcours.

Explorer est parfois plus important que maîtriser.

Récupérer est parfois plus important que se dépasser.

La richesse du graphique vient de la diversité des expériences, pas de leur concentration dans une seule zone.

---

# 4. FLOW ne note jamais l'utilisateur

FLOW ne produit pas de classement.

FLOW ne produit pas de score global.

FLOW ne compare pas les utilisateurs.

Le module aide chacun à comprendre son propre fonctionnement.

Il ne cherche jamais à créer de compétition.

---

# 5. La transparence est obligatoire

L'utilisateur doit toujours pouvoir distinguer :

- ce qu'il a déclaré ;
- ce que MOMENTUM calcule.

Les analyses automatiques ne doivent jamais être présentées comme des vérités.

Lorsque l'algorithme formule une interprétation ou une suggestion, celle-ci doit être clairement identifiable.

---

# 6. Les données restent immuables

Les données utilisateur ne doivent jamais être modifiées automatiquement.

Si les modèles évoluent :

- les calculs évoluent ;
- les visualisations évoluent ;

mais les réponses données par l'utilisateur restent intactes.

Le passé ne doit jamais être réécrit.

---

# 7. La simplicité prime

Le module FLOW doit rester simple à utiliser.

Le temps nécessaire pour compléter une activité doit rester inférieur à quelques secondes.

Si une nouvelle fonctionnalité augmente fortement la complexité sans apporter une réelle valeur, elle ne doit pas être implémentée.

---

# 8. L'intelligence artificielle accompagne

L'intelligence artificielle ne remplace jamais le jugement humain.

Son rôle est :

- d'identifier des tendances ;
- de mettre en évidence des liens ;
- de proposer des pistes de réflexion.

Elle ne décide jamais à la place de l'utilisateur.

---

# 9. Le contexte compte autant que l'activité

Une activité ne peut jamais être interprétée uniquement à partir de ses données physiologiques.

Le contexte fait partie intégrante de l'expérience.

Exemples :

- la fatigue ;
- la météo ;
- le terrain ;
- les personnes présentes ;
- les objectifs ;
- le moment de vie.

Ces éléments pourront progressivement enrichir FLOW.

---

# 10. FLOW appartient à toute la vie

Le module FLOW est né dans le sport.

Mais sa philosophie dépasse largement l'entraînement.

À long terme, il pourra accompagner :

- les activités sportives ;
- les aventures ;
- les moments de bien-être ;
- les découvertes ;
- les projets personnels.

Le langage du module devra rester suffisamment universel pour accompagner l'ensemble des expériences proposées par MOMENTUM.

---

# 11. Chaque évolution doit répondre à une seule question

Avant d'ajouter une nouvelle fonctionnalité, poser systématiquement cette question :

> **Cette évolution aide-t-elle réellement l'utilisateur à mieux comprendre les expériences qui comptent pour lui ?**

Si la réponse est non,

alors cette fonctionnalité n'appartient probablement pas au module FLOW.

---

# 12. Notre engagement

Nous croyons que les performances s'oublient.

Les expériences nous transforment.

FLOW n'existe pas pour mesurer ce que l'utilisateur est capable de faire.

FLOW existe pour l'aider à comprendre pourquoi certains moments deviennent inoubliables.

C'est cette promesse qui guidera toutes les évolutions futures du module.
